Q git push command
A 
git push origin main

Common variants:

git push -u origin main          # first push, sets upstream
git push origin your-branch-name  # push specific branch
git push --force origin main      # force push ⚠
git push --all origin             # all branches
git push --tags origin            # push tags

Typical full workflow:

git add .
git commit -m "your message"
git push origin main
Q From /Users/alexchiomny/eclipse-workspace/test1 to https://github.com/student45667/test1.git
A 
cd /Users/alexchiomny/eclipse-workspace/test1

git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/student45667/test1.git
git push -u origin main

Use a Personal Access Token as password — not your GitHub password.

Q How git push will look with personal token
A 
git push https://<your-token>@github.com/student45667/test1.git main

# Real example:
git push https://ghp_xxxxxxxxxxxxxxxxxxxx@github.com/student45667/test1.git main

Bake token into remote URL (do once):

git remote set-url origin https://ghp_xxxxxxxxxxxxxxxxxxxx@github.com/student45667/test1.git

# Then just:
git push origin main
Q Where to find the token
A 
GitHub → Profile picture → Settings → Developer Settings
→ Personal access tokens → Tokens (classic)
→ Generate new token (classic)
→ Check "repo" scope → Generate → Copy immediately

Direct link: https://github.com/settings/tokens

⚠ GitHub shows the token only once — save it immediately in a password manager.

Q Create bash that streams updated files to repo
A

Polling version (watch_push.sh):

#!/bin/bash

WATCH_DIR="/Users/alexchiomny/eclipse-workspace/test1"
REMOTE="https://ghp_xxxxxxxxxxxxxxxxxxxx@github.com/student45667/test1.git"
BRANCH="main"
INTERVAL=10

cd "$WATCH_DIR" || exit 1
echo "👀 Watching $WATCH_DIR every ${INTERVAL}s..."

while true; do
    if ! git diff --quiet || ! git diff --cached --quiet || [ -n "$(git status --porcelain)" ]; then
        echo "📦 Changes detected at $(date '+%H:%M:%S')"
        git add .
        git commit -m "auto: $(date '+%Y-%m-%d %H:%M:%S')"
        if git push "$REMOTE" "$BRANCH"; then
            echo "✅ Pushed successfully"
        else
            echo "❌ Push failed"
        fi
    else
        echo "⏳ No changes at $(date '+%H:%M:%S')"
    fi
    sleep "$INTERVAL"
done
chmod +x watch_push.sh
./watch_push.sh

Event-driven version with fswatch (recommended):

brew install fswatch

fswatch -o /Users/alexchiomny/eclipse-workspace/test1 | while read; do
    cd /Users/alexchiomny/eclipse-workspace/test1
    git add .
    git commit -m "auto: $(date '+%Y-%m-%d %H:%M:%S')"
    git push https://ghp_xxxxxxxxxxxxxxxxxxxx@github.com/student45667/test1.git main
    echo "✅ Pushed at $(date '+%H:%M:%S')"
done

fswatch fires instantly on file change — better than polling every N seconds.

Q How to make hidden files visible on Finder
A

Keyboard shortcut (fastest):

CMD + SHIFT + .

Terminal (permanent):

# Show
defaults write com.apple.finder AppleShowAllFiles true
killall Finder

# Hide again
defaults write com.apple.finder AppleShowAllFiles false
killall Finder
